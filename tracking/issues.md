## Issue Template

When logging issues, use this format:

### ISSUE-XXX: [Short Title]
- **Date Raised:** DD MMM YYYY
- **Raised By:** Name/Role
- **Phase:** Phase X
- **Severity:** 游댮 Critical / 游리 Major / 游릭 Minor
- **Status:** Open / In Progress / Resolved / Closed
- **Description:** Clear description of the issue
- **Impact:** How this affects the project (timeline, budget, quality, scope)
- **Action Required:** Specific steps to resolve
- **Owner:** Name/Role
- **Due Date:** DD MMM YYYY
- **Resolution Notes:** (filled when resolved)
- **Date Closed:** DD MMM YYYY

## Issue Severity Definitions

| Severity | Definition | Response Time | Escalation |
|----------|------------|---------------|------------|
| 游댮 **Critical** | Blocks milestone completion; major financial/compliance impact | Immediate (same day) | CEO + Board |
| 游리 **Major** | Delays milestone; moderate impact on timeline/budget | 48 hours | Steering Committee |
| 游릭 **Minor** | Low impact; can be resolved in normal workflow | 1 week | Project Manager |

## Issue Management Process

1. **Identification:** Any team member can raise an issue
2. **Logging:** Project Manager logs in this document with unique ID
3. **Assessment:** Assign severity and impact analysis
4. **Assignment:** Allocate owner and due date
5. **Escalation:** Per severity table above
6. **Resolution:** Owner implements action and documents
7. **Verification:** Project Manager confirms resolution
8. **Closure:** Issue marked closed with lessons learned
